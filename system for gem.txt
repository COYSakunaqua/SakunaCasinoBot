# CasinOYS 系統架構與技術白皮書 (System for Gem)
**版本:** V4.3.0 (Dual-Track Readiness Edition)
**時區基準:** HKT (UTC+8)

## 1. 系統當前狀態與雙軌架構
系統目前處於「Discord Bot 維護」與「App 轉型準備」的雙軌並行期。所有底層資料庫邏輯必須兼容未來的 FastAPI 架構。
* **Bot 核心**: `main.py` 負責初始化 `bot.db` 與 `bot.session`，動態載入 Cogs。
* **防護工具**: `utils.helpers.async_db_execute` 負責攔截 Cloudflare 瞬斷並執行退避重試。
* **經濟與排程**: 實作了涓滴救濟、累進富人稅，以及本地無上限線性日息結算。

## 2. 資料庫綱要 (Database Schema - 雙軌過渡版)
### 表格: "Users" (用戶數據)
* `user_id` (TEXT/PK): Discord 用戶 ID (含系統帳戶 'TREASURY')。
* `app_uuid` (UUID/Nullable): 預留給未來 App Supabase Auth 的主鍵。
* `bind_code` (TEXT/Nullable): 預留給未來資料轉移的金鑰。
* `bank` (INTEGER): 銀行存款餘額。
* `daily_lvl` (INTEGER): VIP 等級。
* `last_claim` (TEXT): 最後領取日期 (YYYY-MM-DD)。
* `weekly_profit` (INTEGER) / `weekly_bet_count` (INTEGER) / `debt` (INTEGER) / `current_streak` (INTEGER)。

### 表格: "Events" (賽事數據)
* `event_id` (SERIAL/PK) / `title` (TEXT) / `odds_a`, `odds_b`, `odds_c` (FLOAT)
* `status` (INTEGER): 0 = 待開賽, 2 = 已結算。
* `commence_time` (INTEGER): 開賽 Epoch Timestamp (VAR 鎖定)。
* `winning_choice` (TEXT): 賽果選項 ('A', 'B', 'C')。

### 表格: "Bets" (注單數據)
* `bet_id` (SERIAL/PK) / `user_id` / `event_id` / `choice` / `amount` / `locked_odds`。

## 3. 核心防禦與業務邊界
* **API 額度物理隔離**: The Odds API 僅允許每日 07:00 統一抓取，嚴禁開放給前端或一般玩家觸發。
* **累進稅與線性擴展**: VIP 7 以上的升級與獎勵採「線性固定增長」，拒絕指數膨脹。
* **記憶體極限**: Discord 端必須維持 `gc.collect()` 與 `Yielding`，確保在 100MB RAM 下存活。
